# 주요기능

------------------------------------------------------------
정적 팩토리 메서드
좋은 점:
엔티티와 DTO 간의 변환 로직을 캡슐화:

BoardResponseDto.of(Board board) 메서드를 통해 Board 엔티티를 DTO로 쉽게 변환할 수 있습니다.
이 방식은 컨트롤러나 서비스 레이어에서 변환 로직을 분리하여, 클린하고 유지보수하기 쉬운 코드를 작성할 수 있게 해 줍니다.
댓글 리스트를 함께 처리:

BoardResponseDto가 Comment 리스트를 CommentResponseDto로 변환하여 포함하는 구조로 되어 있어, 게시물과 댓글이 동시에 DTO로 변환되는 점이 효율적입니다.
스트림(stream())을 사용해 Comment 엔티티 리스트를 CommentResponseDto로 깔끔하게 변환하고 있습니다.
빌더 패턴 사용:

CommentResponseDto에 builder() 메서드를 사용해 DTO로 변환하는 부분이 가독성이 좋고, 확장성을 높여줍니다.
추후 CommentResponseDto에 더 많은 필드가 추가되더라도 변환 로직에서 관리가 용이합니다.
